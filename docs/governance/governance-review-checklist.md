# Governance Review Checklist

This checklist is mandatory before approving any domain.

## Required
- Architecture
- DDD
- Clean Architecture
- Aggregate Boundaries
- Entities
- Value Objects
- Repositories
- Policies
- Commands
- Events
- Lifecycle
- Mermaid Diagrams
- Naming Consistency
- Cross References
- Documentation Standards
- Quality Gates
- Architecture Score
- Repository Score
- Traceability
- ADR References
- Approval Status

## Optional
- Additional governance templates
- Extra traceability artifacts
- Supplemental RFC documentation

## Blocked
- Missing required architecture review artifacts
- Missing required governance references
- Unresolved architecture conflicts
- Failure to meet quality gates

## Rejected
- Non-compliant documentation structure
- Incomplete or conflicting traceability
- Unapproved domain modifications
- Unvalidated ADR or ACR linkage

## Usage
- Verify each required item before approving the domain.
- Document the validation results and any exceptions.
- Use the checklist as part of the final governance review.
